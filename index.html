Web Development

My main programming focus so far has been in web development. I was first introduced to html and css concepts at a young age while tinkering around on my myspace page. I was also taught some html in my computer labs in grade school. Fast forward to my studies at Temple University, I began working with html, css, and "vanilla" javascript using the DOM. I was then introduced to many network concepts and protocols. This is when I started working with server-side concepts such as requests, responses, and maintaining state. First, I started with the .NET framework using C#. I was taught all about how to work with controls and handle user input and dynamically display data. I learned how to work with databases and use serialization, as well as security and validation practices. I was then introduced to web API concepts and began working with REST, creating custom controls, as well as SOAP web services. Meanwhile, I began using the Bootstrap framework. This lead to learning alot more about front-end designs and concepts. I started working with some more advanced javasript and learned some jQuery and AJAX. I wanted to begin learning some javascript frameworks and actually ended up diving into Express.js with Node.js. I also learned how to use MailChimp, Heroku, and GitHub Pages. I made some APIs and some sites using these technologies. Finally, I always wanted to learn the Spring Framework using Java. I learned all about how to make REST APIs using Spring web. I then learned all sorts of concepts like spining up servers and deployment using EC2 and S3 in AWS. Going forward, I am going to continue to get better at these sorts of things and my next big thing I want to learn is React!

Rotation Tracker

As part of my Capstone project at Temple, I worked with a team of 5 to plan, design, develop, and test an entire software platform the School of Podiatric Medicine at Temple University. The application helps students, faculty, and administration with the scheduling, editing, and viewing information about medical clerkships that the students must complete as part of their medical training. The Rotation Tracker was made using the .NET framework with C#. We used agile methodologies and git to aid our development. We maintained a trello board and completed weekly SCRUM meetings. We all contributed to each phase of the development process, but my main focus was on the front end design. I used alot of bootstrap and custom CSS, as well as javascript to make a nice looking UI. The process was very rewarding and it was valuable experience to take into industry!

QuizFlash

This was my term project for my class called Server Side Web Application Development. It is a website for studying, where users can create, edit, and study flashcards. It is a Client-Server system with Multi-Tier architecture that includes a complete login system with user registration and security functions. It was made using the .NET framework and used ASMX SOAP and ASP.NET Core REST web APIs that used serialization to send and receive form data in JSON and XML. I utilized stored procedures in T-SQL to perform database functions. For the UI, I used bootstrap and made custom user controls using ASCX and used a few different dynamic data displays. The source code is available on: GitHub

Image Application

For my final project in my class called Practicum in Software Construction, I made a backend API for storing images. This API was made using the Spring Boot framework with java. The application handles requests to store, update, and delete images, with some other features as well. I then spun up an EC2 server on AWS and used an S3 bucket to store all the images. To give server and S3 access to my professor, I assigned IAM roles for him to use. I enjoyed working on the project, and the class in general, because we used popular technologies that are currently being used in industry!
